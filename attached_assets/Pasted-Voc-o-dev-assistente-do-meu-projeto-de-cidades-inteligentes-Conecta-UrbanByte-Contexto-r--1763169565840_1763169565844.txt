### Sobre o usuário MASTER (Superadmin)

Use Supabase (Auth + tabela de controle) para isso:

- Tenha uma tabela de controle, por exemplo `platform_users` (no Supabase, Control Plane), com campos:
  - `id` UUID
  - `email` TEXT (unique)
  - `role` TEXT (ex: 'MASTER', 'TEAM', 'PARTNER')
  - `created_at` TIMESTAMPTZ default now()

- Use o Auth do Supabase como base (usuários logam normalmente), e vincule o usuário autenticado à tabela `platform_users` via `email`.

- Configure uma ENV:
  - `MASTER_ADMIN_EMAIL` → email do superadmin (vou preencher depois no Replit/Supabase, pode deixar como variável de ambiente).

- Na inicialização (ou via script utilitário), se não existir nenhum registro com `role = 'MASTER'` na tabela `platform_users`, crie um registro com:
  - `email = MASTER_ADMIN_EMAIL`
  - `role = 'MASTER'`.

- Proteja a rota do painel:
  - Apenas usuários autenticados no Supabase AND com `role = 'MASTER'` em `platform_users` podem acessar a página do dashboard.
  - Se o usuário não for MASTER, redirecione (por enquanto, pode redirecionar para a home ou exibir mensagem de acesso negado).

### Tipografia e responsividade

- Muito importante: os textos estavam pequenos. Ao converter o HTML:
  - Aumente toda tipografia em 1–2px (ex.: onde era 10px/11px, use 12px/13px).
  - Nenhum texto funcional deve ficar abaixo de ~11px.
  - Mantenha boa hierarquia visual entre título, subtítulo e números.

- Design deve ser totalmente responsivo:
  - Desktop: layout 3 linhas como no protótipo.
  - Tablet: 2 colunas onde fizer sentido.
  - Mobile: cards empilhados, tabelas roláveis horizontalmente se preciso.
